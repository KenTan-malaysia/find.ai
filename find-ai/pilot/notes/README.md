# pilot/notes/

Free-form markdown notes per pilot, mostly from guided walkthrough calls.

## Naming convention

`{pilot-id}-{firstname}.md` — match the ID column in `../tracking.xlsx`.

Examples:
- `01-ahmad.md` — pilot #1, Ahmad
- `02-mei-ling.md` — pilot #2, Mei Ling
- `07-priya.md` — pilot #7, Priya

## Recommended note structure

```markdown
# Pilot 01 — Ahmad Ibrahim
> Date: 2026-04-28 · Channel: Guided Zoom call · 22 min

## Background
Landlord with 2 condos in Mont Kiara, screening tenant #4 this year. Currently uses CCRIS + reference calls, takes 3 days. Open to faster tools.

## Walkthrough observations
- **Step 0 (Landing):** "Don't sign blind" — exhaled, said "yes that's me"
- **Step 2 (LHDN):** Skipped initially, asked "what if my tenant doesn't have one?" — appreciated the skip option
- **Step 3 (Bills):** Picked Upload bill for TNB (had it on email), Account # for Mobile (no bill handy)
- **Step 4 (Score):** "Why is this 67 not 95?" — needed to read the breakdown twice. Once he saw the multiplier explanation, said "ok this is fair"
- **Trust Card:** "I would WhatsApp this to my brother who's also a landlord"

## Verbatim quotes
- "If this saves me 2 days I'd pay RM30 per check"
- "Make the LHDN explanation bigger — I almost skipped it without trying"
- "Can I see what BAD scores look like? I want to see the contrast"

## Recommendations heard
- Toggle to view what a low-score tenant looks like (T5 from 100-user audit — confirmed real)
- WhatsApp share button on Trust Card (T7 from audit — confirmed real)
- Pricing: per-screen RM20-30 lands well, subscription not preferred ("only screen 4-6 times a year")

## NPS asked verbally: 9
## Use intent: Yes, immediately for next vacancy
```

## How Zeus reads these

When asked "summarize pilot feedback" or "what are common themes", Zeus reads all `.md` files in this folder + `responses/` CSVs, clusters quotes/observations into themes, and surfaces patterns.

## Privacy note

Notes contain personal observations and verbatim quotes. Don't commit to public git unless anonymized. Repo is currently private — safe for now.
